用Github pages、freenom和vercel免费建立个人博客

发表于 2020-08-08 阅读次数：

从中午忙到现在，通过不断探索，终于把网站搭好了，记录一下简要的过程，总结经验。

总体思路

在Github上建立个人仓库，用来存放网站的主要内容。

笔者使用了Gridea用来管理博客内容。（非必需）

在freenom上申请域名。

用vercel托管仓库并负责域名解析。

注意，最好自备梯子。

具体步骤

1.注册github账号并建立仓库

首先，需要在 https://github.com/ 上注册一个账号。注意：邮箱请不要使用QQ邮箱，否则会影响到之后vercel的使用。如果验证码无法加载，或许需要使用VPN。

接着，建立一个仓库（Repository），仓库的命名必须为 username.github.io，如AgentC137.github.io，个人博客的内容将存放在里面。

点击右上角的头像，依次选择Settings– Developer settings–Personal access tokens，建立一个令牌（token），“Note”填写项目简介（可随意），在“Select scopes”项目的“repo”前打勾，然后点创建。创建了一个令牌（token）后，会看到一长串随机生成的字符，也即是令牌本体，将它复制到本文文档中，或者其他你之后还能找到的文件里暂存。

（注意，此时有两个选择，一个是自己手动寻找博客模板，一个是使用傻瓜式软件Gridea，笔者在这里以Gridea为例。）

2.配置Gridea

（1）下载Gridea（https://gridea.dev/），安装。

（2）在“远程”界面配置相关内容。

“平台”请选择Github Pages。
“域名”先空着，之后有用。
“仓库名称”填写之前创建的名称，即username.github.io（把username换成自己的）
“分支”填master
“仓库用户名”填注册用户名
“邮箱”填写注册邮箱。
“令牌”填写上一节中得到的令牌（token，即一长串字符），注意不要错误填写成密码。
“CNAME”暂时不填。

在做完这些后，点击“检测远程连接”，查看是否有问题。若有问题，请点击左下角的github图标去issue区浏览有无相似问题。

（3）在文章界面试着写一个hello world（可选项）

（4）挑选一个主题并预览

（5）点击同步。

3.在freenom或其他域名申请网页上申请域名。

freenom（https://www.freenom.com/）的域名一年免费，并且服务器在国外，无须备案。申请好域名之后，返回Gridea，在“域名”“CNAME”中填写域名，不需要带https字样，如：agentc137.cf 。

4.用vercel托管仓库，之后在freenom上进行域名解析。

https://vercel.com/dashboard 可以使用github注册，如果github的邮箱是QQ邮箱，则会提示错误。注册后将仓库托管，托管的仓库名叫Project。进入其中，点击settings–domains，添加自己的网站地址，点击nameservers，将给出的两行
ns1.vercel-dns.com
ns2.vercel-dns.com
写进freenom的DNS解析中。

配置工作大功告成。静等DNS生效，尝试访问网站。Hello world！

写在后面

一开始我尝试用cloudflare进行域名解析，无奈速度太慢，不挂梯子很难访问且影响排版。之后又尝试了其他的一些方法，都不理想，所幸朋友说可以使用vercel，终于成功。探索与“折腾”可以带来很大乐趣，成功之后的满足感也叫人愉快。

探索是无止境的，这个教程也是一个初步版本，事实上还有许许多多的内容可以发掘，例如Gridea自带的主题少得可怜，对markdown的支持也不算很好，我之后打算换换主题。另外，这个网站也是静态网站，暂时没有评论区，之后可以研究着部署评论功能。
